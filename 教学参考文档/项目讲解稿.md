# Real FPS — 项目讲解稿

> 一个基于计算机视觉的实弹射击训练系统  
> 讲稿用时：约 8~12 分钟

---

## 一、项目概述（约 1 分钟）

**一句话：**  
用摄像头识别人体 → 云台自动跟踪瞄准 → 鼠标/自动开火 → 得分 + 特效反馈

**核心流程：**

```
摄像头采集 → YOLO 人体姿态估计 → 准星对准目标
    → 开火检测 → 串口发送角度 → 舵机云台转动
    → 得分计算 → UDP 通知 UI → 粒子特效 + 音效
```

**三个游戏模式：**
| 模式 | 自动瞄准 | 自动开火 | 说明 |
|------|---------|---------|------|
| A | ✗ | ✗ | 纯手动，练枪法 |
| B | ✓ | ✗ | 辅助瞄准，手动开火 |
| C | ✓ | ✓ | 全自动索敌击杀 |

## 二、计算机视觉管线（约 2 分钟）

### 2.1 摄像头与共享内存

```
USB 摄像头 → camera_share.py (FastAPI 服务)
    ├── 全分辨率 1280×720 → HTTP /snapshot → Pygame UI 显示
    └── 缩略图 640×480 → 共享内存 → YOLO 推理
```

**亮点：** 用 `multiprocessing.shared_memory` 实现了**零拷贝帧传输**，YOLO 推理直接从共享内存读原始帧，不走 HTTP 序列化。

### 2.2 YOLOv8n-pose 人体跟踪

```
YOLO 模型 → model.track() → 17 个关键点
    ├── 头部框：从耳朵、眼睛、鼻子关键点推算
    ├── 身体四边形：从肩膀、髋部关键点连接
    └── 击中判定：射线法判断准星是否落在框内
```

- 每个目标分配稳定跟踪 ID（`persist=True`）
- 只跟踪 `class=0`（person），排除误检
- 推理在缩略图 (640×480) 上进行，坐标再映射回全分辨率

### 2.3 LBPH 人脸识别

```
YOLOv11n-face 检测人脸区域
    → 裁剪对齐 → LBPH 预测
    → 显示人名标签在目标框上方
```

- 支持注册已知人员（通过 `register.py` 拍照录入）
- 未识别的标记为 "Unknown"

## 三、云台控制（约 1.5 分钟）

### 3.1 硬件架构

```
PC (Python)                STM32F103C8             舵机
┌──────────┐   串口     ┌──────────────┐    PWM    ┌──────┐
│ main.py  │ ────────→ │  USART2 接收  │ ────────→ │ SG90 │
│          │ "yaw,pitch"│  ParseAngle   │  TIM1/2   │ × 2  │
└──────────┘            └──────────────┘           └──────┘
```

- 协议格式：`"yaw,pitch\n"`，支持**浮点数**（如 `"12.5,-30.0\n"`）
- 波特率 115200，每帧发送一次
- STM32 端用逐字节中断接收 + 自定义浮点解析（不依赖 `sscanf %f`）

### 3.2 自动瞄准算法（AutoAimPID）

```
每 skip+1 帧判断一次方向
    → 方向锁定计数（防噪声翻转）
    → 把修正量均摊到后续每帧输出
```

**特点：**
- 死区 + 滞后（防止目标在准星附近来回抖动）
- 方向锁定（连续 N 帧反向才切换，过滤检测噪声）
- 均摊输出（避免舵机阶跃造成的机械震动）

│  └──────────────┘  └──────────────────┘ │
└─────────────────────────────────────────┘

### 4.2 关键 UI 元素

**HUD 面板（左上角）：**
- 得分 + 弹药数值（带投影，不发虚）
- 弹药进度条（>15 绿、5~15 渐变、<5 红）
- 目标数 / FPS 同排显示
- 底部状态栏（模式 + 串口状态）

**B-scope 雷达（右上角）：**
- 矩形扫描雷达，目标水平方位 + 距离映射
- 发光边框 + 扫描线尾迹特效
- 目标点闪烁，锁定变黄色

**准星：**
- 四方向线段 + 霓虹双色
- 开火时脉冲放大再收缩

**命中反馈：**
- 全屏闪白（300ms 淡出）
- 得分弹出（上移 80px，不挡准星）
- 击杀粒子爆发（爆头=红色，身体=绿色）
- 击杀音效 (`head_shot.mp3`)

**3D 枪械（右下角）：**
- `.obj` 模型加载
- 呼吸动画、开火后坐、检视旋转、换弹序列

### 4.3 通信架构

```
状态通道（低频）：state.json（原子写入）
    └── 写入 → tmp 文件 → os.replace() 原子移动

事件通道（实时）：UDP 本地广播
    ├── 开火事件 → 端口 8099
    ├── 击杀事件 → 端口 8099
    └── 换弹完成 → 端口 8098
```

## 五、嵌入式端（约 1 分钟）

### STM32 下位机

**芯片：** STM32F103C8T6 (Cortex-M3, 72MHz)

**外设：**
| 外设 | 用途 |
|------|------|
| TIM1 CH1 | Yaw 轴舵机 PWM (50Hz) |
| TIM2 CH1 | Pitch 轴舵机 PWM (50Hz) |
| USART2 | 接收 PC 角度指令 (115200bps) |

**软件架构：**
```
USART2 中断接收
    → 逐字节拼接行缓冲区
    → '\n' 触发解析 → 角度限幅 → PWM 更新
```

**计算公式：**
```
脉冲宽度 = 150 + angle × 100 ÷ 90
  -90° → 50 ticks  (500μs)
   0°  → 150 ticks (1500μs)
  +90° → 250 ticks (2500μs)
```

**自主实现：**
- 自定义 `ParseFloat()` 替代 `sscanf("%f")`，避免 newlib-nano 不支持浮点格式的问题
- 角度限幅 `±90°`，防舵机机械碰撞

## 七、演示流程建议

```
1. 启动 start.py → 选择方案 A（手动模式）
   → 展示：摄像头画面、HUD、准星、雷达
   
2. 移动鼠标 → 展示：视角控制、目标框 + 人名标签
   
3. 开火（左键）→ 展示：后坐力、准星脉冲、弹药减少、进度条变化
   
4. 切换到方案 C（全自动）
   → 展示：云台自动搜索、锁定、跟踪、开火
   
5. 击杀目标 → 展示：粒子爆发、得分弹出、击杀音效、击杀通知
   
6. 按 P 暂停 → 展示：PAUSED 覆盖层
   
7. 串口断开 → 展示：底部状态栏变为 ERROR
```
