# Real FPS — 项目讲解稿

> 一个基于计算机视觉的实弹射击训练系统  
> 讲稿用时：约 8~12 分钟

---

## 一、项目概述（约 1 分钟）

**一句话：**  
用摄像头识别人体 → 云台自动跟踪瞄准 → 鼠标/自动开火 → 得分 + 特效反馈

**核心流程：**

```
摄像头采集 → YOLO 人体姿态估计 → 准星对准目标
    → 开火检测 → 串口发送角度 → 舵机云台转动
    → 得分计算 → UDP 通知 UI → 粒子特效 + 音效
```

**三个游戏模式：**
| 模式 | 自动瞄准 | 自动开火 | 说明 |
|------|---------|---------|------|
| A | ✗ | ✗ | 纯手动，练枪法 |
| B | ✓ | ✗ | 辅助瞄准，手动开火 |
| C | ✓ | ✓ | 全自动索敌击杀 |

---

## 二、计算机视觉管线（约 2 分钟）

### 2.1 摄像头与共享内存

```
USB 摄像头 → camera_share.py (FastAPI 服务)
    ├── 全分辨率 1280×720 → HTTP /snapshot → Pygame UI 显示
    └── 缩略图 640×480 → 共享内存 → YOLO 推理
```

**亮点：** 用 `multiprocessing.shared_memory` 实现了**零拷贝帧传输**，YOLO 推理直接从共享内存读原始帧，不走 HTTP 序列化。

### 2.2 YOLOv8n-pose 人体跟踪

```
YOLO 模型 → model.track() → 17 个关键点
    ├── 头部框：从耳朵、眼睛、鼻子关键点推算
    ├── 身体四边形：从肩膀、髋部关键点连接
    └── 击中判定：射线法判断准星是否落在框内
```

- 每个目标分配稳定跟踪 ID（`persist=True`）
- 只跟踪 `class=0`（person），排除误检
- 推理在缩略图 (640×480) 上进行，坐标再映射回全分辨率

### 2.3 LBPH 人脸识别

```
YOLOv11n-face 检测人脸区域
    → 裁剪对齐 → LBPH 预测
    → 显示人名标签在目标框上方
```

- 支持注册已知人员（通过 `register.py` 拍照录入）
- 未识别的标记为 "Unknown"

## 三、云台控制（约 1.5 分钟）

### 3.1 硬件架构

```
PC (Python)                STM32F103C8             舵机
┌──────────┐   串口     ┌──────────────┐    PWM    ┌──────┐
│ main.py  │ ────────→ │  USART2 接收  │ ────────→ │ SG90 │
│          │ "yaw,pitch"│  ParseAngle   │  TIM1/2   │ × 2  │
└──────────┘            └──────────────┘           └──────┘
```

- 协议格式：`"yaw,pitch\n"`，支持**浮点数**（如 `"12.5,-30.0\n"`）
- 波特率 115200，每帧发送一次
- STM32 端用逐字节中断接收 + 自定义浮点解析（不依赖 `sscanf %f`）

### 3.2 自动瞄准算法（AutoAimPID）

```
每 skip+1 帧判断一次方向
    → 方向锁定计数（防噪声翻转）
    → 把修正量均摊到后续每帧输出
```

**特点：**
- 死区 + 滞后（防止目标在准星附近来回抖动）
- 方向锁定（连续 N 帧反向才切换，过滤检测噪声）
- 均摊输出（避免舵机阶跃造成的机械震动）

### 4.2 HUD 面板（左上角）— B 负责

**布局：**
```
┌──────────────────────┐
│ 得分  100            │  ← 标签(小)+数值(大)，同排
│ 弹药  30             │  ← 弹药不足10变红
```

**技术实现：**
- `ui/hud.py` 中的 `HUD.render()` 方法
- 字体使用自定义 `.ttf` 字体，通过 `ui/assets.py` 的 `get_font()` 统一管理
- **投影文字**替代叠层发光——黑色偏移 2px 渲染一次，主文字一次，清晰锐利
- 弹药进度条：根据剩余量动态变色，`>15` 绿色、`5~15` 黄到橙渐变、`<5` 红色

- 全部标签中文："得分"、"弹药"、"目标"、"模式"、"串口"
- 底部状态栏：半透明暗色背景条 + 投影文字，显示当前模式和串口状态

### 4.3 B-scope 雷达（右上角）— B 负责

```
    N
  ┌──────┐
W │ 扫   │ E   ← 矩形扫描区域，非圆形
  │ 描线 │
  └──────┘        · X 轴 = 水平方位
    S             · Y 轴 = 距离（近在下，远在上）
```

**技术实现：**
- `ui/radar.py` 中的 `Radar.render()` 方法
- **发光边框**：外层半透明绿框（光晕）+ 内层 4px 亮绿边框，带圆角
- **左侧距离刻度**：三条短线标记距离参考
- **扫描线尾迹**：`deque(maxlen=10)` 保存最近 10 帧位置，从旧到新依次变亮绘制
- **目标标记**：存活目标绿色圆点闪烁，锁定变黄色，击杀显示绿色叉号
- B 同学还可以补充：网格线透明度、目标点大小随距离变化等细节

### 4.4 准星与命中反馈 — B 负责

**准星（屏幕中心）：**
- 四方向线段：内圈 20px → 外圈 32px
- 双色渲染：青色主线 `(0,255,220)` + 品红辅线 `(255,0,160)`
- **开火脉冲**：击中瞬间内圈扩至 30px、外圈扩至 44px，150ms 内平滑恢复

**命中反馈（`ui/effects.py`）：**
- **全屏闪白**：`HitFlash` 类，300ms 白色淡出
- **得分弹出**：`ScorePopup` 类，如 "+50 headshot"，上移 80px 不挡准星
  - 三段动画：200ms 淡入 → 1000ms 停留 → 400ms 淡出
- **击杀粒子爆发**：`ParticleBurst` 类，25 个粒子向四周飞散，800ms 消失
  - 爆头 → 红色/橙色粒子
  - 身体击杀 → 绿色粒子
- **击杀音效**：播放 `music/sounds/head_shot.mp3`

### 4.5 3D 枪械（右下角）

- `ui/gun_view.py` + `ui/obj_loader.py`，使用 **PyOpenGL** 渲染 `.obj` 模型
- 预渲染 10 帧开火动画序列，开火时按后坐力进度播放对应帧
- 呼吸动画：正弦波 3px 上下微动
- 检视动画（按 I 键）：30 帧旋转到侧视角再回正
- 换弹动画（按 R 键）：60 帧序列（退匣 → 装弹 → 拉栓），配合音效时序
- 枪口火焰：BLEND_ADD 混合模式叠加

### 4.6 击杀通知 — B 负责

- `ui/kill_feed.py`，左下角逐条显示
- 每条持续 3 秒，最后 500ms 淡出消失
- 最多同时显示 4 条
- 格式：`目标名  部位  +得分`

### 4.7 通信架构

```
状态通道（低频）：state.json（原子写入）
    └── main.py 每帧写入 → tmp 文件 → os.replace() 原子移动
    └── UI 每 50ms 轮询读取

事件通道（实时）：UDP 本地广播
    ├── 开火事件 → 端口 8099
    ├── 击杀事件 → 端口 8099（含 hit_zone, target_id, target_name）
    └── 换弹完成 → 端口 8098（UI → main.py）
```


## 五、键鼠控制 — C 的工作（约 2 分钟）

> 这部分由 **C 同学** 上台讲解。

### 5.1 鼠标控制模块

**文件：** `mouse.py` — `MouseListener` 类

**架构：**
```
pynput.mouse.Listener (后台线程)
    → 采集鼠标绝对位移 → 存入线程安全队列
    → get_delta() 由主循环每帧取出并累加
    → 死区过滤 + 灵敏度缩放 → 返回 (dx, dy, left_pressed)
```

**关键特性：**

| 特性 | 实现 | 说明 |
|------|------|------|
| 死区 | `deadzone=2` | 微小移动被过滤，防止准星抖动 |
| 灵敏度 | `sensitivity=1.0` | 可调系数，乘到原始位移上 |
| 平滑 | `smooth_window=3` | 滑动平均滤波，3 帧窗口 |
| 鼠标回中 | `center_lock=True` | `SetCursorPos` 持续拉回屏幕中心 |
| 回中节流 | `_center_interval=0.05` | 最小 50ms 间隔，避免过频 API 调用 |

**生命周期：**
```
__init__() → 启动 pynput 监听器（后台线程常驻，但不采集）
start()    → 收到 GAME_START，开始采集 + 鼠标回中
pause()    → 暂停采集
resume()   → 恢复采集 + 回中
stop()     → 完全停止监听器
```

### 5.2 键盘快捷键

| 按键 | 功能 | 实现位置 |
|------|------|---------|
| **鼠标左键** | 开火 | `mouse.py` → `_on_click()` 检测左键按下 |
| **P** | 暂停/继续 | `mouse.py` → `emitter.emit("GAME_PAUSE/CONTINUE")` |
| **Esc** | 退出游戏 | `mouse.py` → `emitter.emit("GAME_OVER")` |
| **R** | 换弹 | `mouse.py` → `emitter.emit("GAME_RELOAD")` |
| **左键（点按）** | 锁定/解除目标 | `mouse.py` → `emitter.emit("GAME_LOCK")` |
| **↑/↓** | BGM 音量 ±10% | `ui/core.py` → `pygame.mixer.music.set_volume()` |
| **I** | 检视枪械 | `mouse.py` → `emitter.emit("GAME_INSPECT")` |

### 5.3 视角控制公式

```
手动模式：
    v_x += dx × YAW      # YAW = 0.1
    v_y += dy × PITCH    # PITCH = 0.1
    
    v_x = clamp(v_x, -90, 90)   # 云台水平限位
    v_y = clamp(v_y, -60, 60)   # 云台俯仰限位
```

- `v_x, v_y` 是当前云台角度（度），持续累积
- 鼠标位移 `dx, dy` 乘灵敏度后累加到角度上
- 再通过串口发送给 STM32，驱动舵机转动

### 5.4 模式切换

三种模式由 `start.py` 的启动参数控制：

```
A 模式：纯手动
    BOX=False  LOCK=False  AUTO_FIRE=False  → 纯练枪法
    
B 模式：辅助瞄准
    BOX=True   LOCK=True   AUTO_FIRE=False  → 雷达+框可见，自动瞄准，手动开火
    
C 模式：全自动
    BOX=True   LOCK=True   AUTO_FIRE=True   → 自动搜索+锁定+开火
```

C 模式下自动搜索逻辑：
1. 无目标时，云台从当前位置向 +90° 匀速扫描（0.8°/帧）
2. 检测到目标 → 自动锁定最近的活目标
3. 锁定后 AutoAimPID 持续跟踪
4. 舵机到限位仍追不上 → 自动放弃目标（加入黑名单，不计分）
5. 目标丢失/击杀 → 自动重新锁定下一个最近目标




## 七、嵌入式端（约 1 分钟）

### STM32 下位机

**芯片：** STM32F103C8T6 (Cortex-M3, 72MHz)

**外设：**
| 外设 | 用途 |
|------|------|
| TIM1 CH1 | Yaw 轴舵机 PWM (50Hz) |
| TIM2 CH1 | Pitch 轴舵机 PWM (50Hz) |
| USART2 | 接收 PC 角度指令 (115200bps) |

**软件架构：**
```
USART2 中断接收
    → 逐字节拼接行缓冲区
    → '\n' 触发解析 → 角度限幅 → PWM 更新
```

**计算公式：**
```
脉冲宽度 = 150 + angle × 100 ÷ 90
  -90° → 50 ticks  (500μs)
   0°  → 150 ticks (1500μs)
  +90° → 250 ticks (2500μs)
```

**自主实现：**
- 自定义 `ParseFloat()` 替代 `sscanf("%f")`，避免 newlib-nano 不支持浮点格式的问题
- 角度限幅 `±90°`，防舵机机械碰撞

## 九、演示流程建议

```
1. 启动 start.py → 选择方案 A（手动模式）
   → B 展示：摄像头画面、HUD、准星、雷达
   
2. 移动鼠标 → C 展示：视角控制、目标框 + 人名标签
   
3. 开火（左键）→ B+C 展示：后坐力、准星脉冲、弹药减少、进度条变化
   
4. 切换到方案 C（全自动）
   → D 展示：云台自动搜索、锁定、跟踪、开火
   → 舵机限位 → 展示自动放弃
   
5. 击杀目标 → B 展示：粒子爆发、得分弹出、击杀音效、击杀通知
   
6. 按 P 暂停 → B 展示：PAUSED 覆盖层
   
7. 按 R 换弹 → C 展示：换弹动画 + 音效时序
   
8. 串口断开 → B 展示：底部状态栏变为 ERROR
```
